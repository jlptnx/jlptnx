## Design Principles

### 1. Clarity Over Cleverness
Every element should immediately communicate its purpose. JLPT learners are already dealing with complex materialâ€”the interface should be invisible.

### 2. Momentum, Not Guilt
Visual language should celebrate progress, not shame failure. Missed days happen; the design should encourage recovery, not punishment.

### 3. Compact Information Density
Show what matters at a glance. Streaks, pod activity, and check-in status should be scannable in seconds.

### 4. Japanese-Friendly Typography
Support mixed Japanese/English text beautifully. Pod names, notes, and content may contain kanji, hiragana, and katakana.

### Signature Animations

1. **Streak Increment** - Number counts up with fire particle burst
2. **Check-in Success** - Checkmark draws in, confetti for milestones
3. **Pod Activity** - New items slide in from bottom with fade
4. **Sheet Open** - Slides up with slight bounce at rest

## Accessibility

### Requirements

- WCAG 2.1 AA compliance minimum
- Minimum touch target: 44x44px
- Color contrast ratio: 4.5:1 for normal text, 3:1 for large text
- Support for reduced motion preference
- Screen reader announcements for dynamic content
- Keyboard navigation support

### Focus States

All interactive elements must have visible focus indicators:
- 2px primary-500 ring
- 2px offset from element
